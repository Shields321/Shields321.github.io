Overview of the AES Algorithm

AES (Advanced Encryption Standard) is a symmetric encryption algorithm widely used to secure data. In this implementation, we focus on using a 128-bit key for encryption and decryption. Below is an explanation of the AES algorithm, broken down into its key steps:

1. Key Expansion

The first step in the AES algorithm is key expansion. The original 128-bit key is expanded into an array of key schedule words, which are used in later rounds. This key schedule contains multiple 32-bit words derived from the original key, ensuring the key material varies across the encryption rounds. The expansion uses a process of rotating the key and applying the Rijndael S-box to generate new key material.

2. Initial Round - AddRoundKey

The initial round begins by XORing the plaintext block with the first 128-bit round key derived from the key expansion. This step is called the AddRoundKey operation. It ensures that the input data is mixed with the key material before proceeding to the main rounds.

3. Main Rounds - SubBytes, ShiftRows, MixColumns, AddRoundKey

AES uses several operations in its main rounds. These rounds are repeated multiple times (10 rounds for AES-128). The operations are as follows:

  • SubBytes: This step substitutes each byte in the state matrix with its corresponding value from the S-box. The S-box is a pre-defined substitution table that provides non-linearity in the encryption process.
  • ShiftRows: The rows of the state matrix are shifted cyclically. The first row remains unchanged, the second row is shifted by one byte, the third row by two bytes, and the fourth row by three bytes.
  • MixColumns: This operation mixes the data in each column of the state matrix. It applies a matrix multiplication in the finite field to provide diffusion, ensuring that the output of each column depends on the entire input block.
  • AddRoundKey: After performing the SubBytes, ShiftRows, and MixColumns steps, the current state is XORed with the next round key from the key schedule.

4. Final Round - SubBytes, ShiftRows, AddRoundKey

The final round is similar to the main rounds, except it does not include the MixColumns step. It consists of:

  • SubBytes: As before, each byte in the state matrix is substituted using the S-box.
  • ShiftRows: The rows of the state matrix are again shifted cyclically.
  • AddRoundKey: Finally, the last round key is XORed with the state matrix, completing the encryption process.

5. Output the Ciphertext

After the final round, the state matrix is now the ciphertext, which is output as the result of the AES encryption process. The ciphertext is the encrypted version of the original plaintext.

Decryption

Decryption is the reverse of the encryption process. It uses the same steps but in reverse order, with inverse operations for each step (e.g., InverseSubBytes, InverseShiftRows, and InverseMixColumns). The round keys are also applied in reverse order, starting from the last key in the key schedule.
